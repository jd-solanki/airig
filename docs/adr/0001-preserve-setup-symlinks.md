# Preserve setup symlinks

Project Instruction Files are first-class AI Setup artifacts like skills, agents, and commands, with the distinction that their source files live at the root of `.ai/`. They can be shared across agent-recognized filenames by making one `.ai/` artifact a symlink to another, such as `.ai/CLAUDE.md -> AGENTS.md`. Authors are responsible for creating each source instruction artifact in `.ai/`, whether as separate files or symlinks; `ohmyai` does not synthesize `.ai/CLAUDE.md` from `.ai/AGENTS.md`. `ohmyai` preserves those symlinks through publish and install instead of materializing duplicate files, because the symlink records the author's intent that two Instruction Targets share the same guidance rather than merely having matching content at one point in time.
